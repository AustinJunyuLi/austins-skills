# Code Review

## Overview

Front door for software review. This skill covers direct code review, packaged external review, and high-stakes second-opinion workflows for code or design.

## Use This For

- reviewing a diff
- auditing a codebase
- looking for bugs or regressions
- packaging code for external reviewers
- adversarial or panel-style design review

## Do Not Use This For

- reviewing papers or grants: use `research-review`
- planning implementation work: use `superpowers:writing-plans`
- code-review process skills that already live in the superpowers family

## Explicit Superpowers Boundary

Keep these process skills separate and explicit:

- `superpowers:requesting-code-review`
- `superpowers:receiving-code-review`
- `superpowers:verification-before-completion`

## Internal Routing

- local code review
- external packaging workflows
- panel-style second opinions

## Operating Rule

If the target artifact is code rather than research prose or slides, start here.
